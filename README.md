# Приложение «Заметки»

## Описание проекта

Программный проект «Заметки» представляет собой приложение для создания, хранения и структурирования текстовой информации с возможностью объединения отдельных заметок в книги, назначения ярлыков и экспорта в формат PDF.

Система предназначена для организации персонального хранилища данных пользователя: идей, учебных материалов, рабочих записей, конспектов и иной текстовой информации.

Приложение обеспечивает:

- удобную навигацию по хранилищу;
- быстрый поиск по ключевым словам и ярлыкам;
- структурирование заметок по тематическим книгам или сборникам.

---

## Основные реализуемые инструменты

В рамках проекта предполагается реализация следующих программных средств:

1. Создание новой заметки
2. Редактирование текста
3. Удаление заметки
4. Автоматическое сохранение
5. Форматирование текста (жирный, курсив, заголовки)
6. Создание списков
7. Вставка изображений
8. Добавление гиперссылок
9. Назначение ярлыков
10. Редактирование ярлыков
11. Фильтрация по ярлыкам
12. Поиск по тексту
13. Объединение заметок в книгу
14. Изменение порядка заметок в книге
15. Просмотр книги в режиме чтения
16. Экспорт заметки в PDF
17. Экспорт книги в PDF
18. Импорт заметок
19. Сортировка по дате создания
20. Сортировка по дате изменения
21. Закрепление заметки
22. Архивирование
23. Восстановление из архива
24. Тёмная и светлая темы интерфейса

**Общее количество реализуемых средств — не менее 20.**

## Модули программного проекта

### 1. Модуль пользовательского интерфейса

Обеспечивает взаимодействие пользователя с системой.

Интерфейс включает:

- панель навигации по заметкам и книгам;
- область редактирования текста;
- панель ярлыков;
- строку поиска;
- панель инструментов форматирования.

---

### 2. Модуль управления заметками

Отвечает за создание, хранение, редактирование и удаление заметок.

Обеспечивает:

- автоматическое сохранение изменений;
- хранение метаданных (дата создания, дата изменения, статус закрепления).

---

### 3. Модуль управления книгами

Реализует объединение заметок в логические сборники (книги).

Обеспечивает:

- добавление и удаление заметок из книги;
- изменение порядка их следования;
- формирование структуры содержания.

---

### 4. Модуль ярлыков и категорий

Позволяет:

- создавать, редактировать и удалять ярлыки;
- выполнять фильтрацию и быстрый поиск заметок по назначенным меткам.

---

### 5. Модуль поиска

Реализует полнотекстовый поиск по:

- содержимому заметок;
- названиям книг;
- ключевым словам;
- ярлыкам;
- дате создания.

---

### 6. Модуль форматирования текста

Обеспечивает базовые средства редактирования:

- заголовки;
- списки;
- выделение текста;
- вставку изображений;
- вставку ссылок.

Может поддерживать разметку Markdown или встроенный текстовый редактор.

---

### 7. Модуль экспорта в PDF

Отвечает за формирование документа в формате PDF.

Обеспечивает корректное отображение:

- структуры текста;
- изображений;
- форматирования  
  как для отдельной заметки, так и для всей книги.

---

### 8. Модуль импорта данных

Позволяет загружать заметки из текстовых файлов или других поддерживаемых форматов.

---

### 9. Модуль хранения данных

Реализует работу с базой данных или файловой системой.

Обеспечивает:

- сохранность информации;
- резервное копирование;
- восстановление данных.

---

### 10. Модуль синхронизации (при расширении функциональности)

Обеспечивает синхронизацию заметок между устройствами через сервер или облачное хранилище.

---

### 11. Модуль управления проектом и настройками

Реализует:

- создание нового хранилища;
- резервное копирование;
- настройку интерфейса (тема, шрифт, язык);
- обработку ошибок;
- системные уведомления.

---

## Используемые технологии

Для реализации могут применяться современные языки программирования и фреймворки в зависимости от платформы разработки (веб, десктоп или мобильная среда).

### Возможные технологии:

- Веб-версия — JavaScript / TypeScript с использованием React, Vue или аналогичных фреймворков
- Десктоп-приложение — C# (.NET), Java или Electron
- Мобильная версия — Flutter, React Native или нативная разработка

Для хранения данных может использоваться:

- SQLite
- PostgreSQL
- встроенная файловая система

---

## Ожидаемый результат

В результате разработки будет создано приложение для ведения и структурирования заметок с возможностью:

- объединения их в книги;
- назначения ярлыков для быстрого поиска;
- экспорта данных в формат PDF.

Программа должна обеспечивать:

- удобный интерфейс;
- стабильное хранение данных;
- расширяемую архитектуру.

Архитектура проекта должна быть модульной, что обеспечит возможность дальнейшего развития функциональности, включая:

- синхронизацию;
- совместную работу пользователей;
- расширенные средства форматирования.
